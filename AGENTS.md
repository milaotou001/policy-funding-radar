# AI 协作指引 — 政策资金流向雷达

> Claude 和 Codex 打开项目时先读本文件。

---

## 统一口令

用户对任何工具说：**"按工作流继续"**

工具收到后自动执行：

1. 优先读取 STATUS.md
2. 如果没有 STATUS.md，再读取 CURRENT.md
3. 读取 AGENTS.md（本文件）
4. 必要时读取 PRD.md / TODO.md / README.md
5. 根据 STATUS.md / CURRENT.md 中的当前任务继续
6. 允许读取和修改完成当前任务所需的相关文件
7. 禁止做当前任务范围外的功能
8. 禁止扩大 MVP
9. 完成后简短更新 STATUS.md
10. 阶段切换时才更新 CURRENT.md（如存在）

---

## 两套状态文件

| 文件 | 用途 | 更新频率 |
|------|------|---------|
| **STATUS.md** | 日常开发状态 | 每完成一个任务更新 |
| **CURRENT.md** | 阶段交接控制 | 只在阶段切换时更新 |

---

## 工具分工

| 工具 | 做什么 | 不做什么 |
|------|--------|---------|
| **ChatGPT** | 白天收集想法、出启动材料 | 不创建项目、不写代码 |
| **Claude** | 需求澄清、MVP确认、项目初始化、阶段审查、数据产出 | 标准模式下不逐任务介入实现 |
| **Codex** | 技术复核、连续实现（标准模式下承担中） | 不重新定义需求、不扩大MVP |
| **Task Master** | 把PRD拆成任务列表（重型模式） | 不写代码、不参与需求 |

---

## Codex 日常开发规则

Codex 可以连续执行一组明确任务，但必须：
1. 每次只做一个任务
2. 每完成一个任务就 Git commit
3. 只简短更新 STATUS.md（不长篇更新 CURRENT.md）
4. 不扩大 MVP
5. 不做任务清单以外的功能
6. 技术复核时不写代码，只输出报告

---

## Claude 审查规则

Claude 不需要逐个小任务审查。只在以下情况审查：
- 一个任务组完成
- Codex 明确遇到风险
- 用户要求审查
- 阶段结束前

---

## 核心约束

### MVP-0 范围
- 全国 10 个产业标签（新能源、数字经济、人工智能、新能源车、制造业、低空经济、生物医药、新材料、半导体/芯片、储能）
- 三层对比：原文高亮 + 产业标签表 + 投资观察卡片
- 单页纵向滚动，桌面优先，响应式布局
- 数据来源：静态 JSON (`data/industries.json`)

### MVP-0 不做
- 各省数据（只做全国）
- AI API 对接（静态数据代替）
- PDF 上传解析
- 用户上传原文
- 地区/领域/对比模式筛选器
- 变化排名视图
- 多页面路由
- ETF/股票推荐
- 后端服务

### Claude 主力规则
- 同一个问题问 3 次没解决 → 更新 STATUS.md 交 Codex
- 每次会话结束更新 STATUS.md
