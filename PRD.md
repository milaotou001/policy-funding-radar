# 产品需求文档 — 政策资金流向雷达

> 创建日期：2026-05-31
> 版本：v1.0（MVP-0）

---

## 1. 项目背景

### 解决的问题

投资研究者和产业分析人员需要理解国家五年规划中产业政策的变化方向——哪些产业被新增、强化、弱化、工程化、监管化。传统的"读报告"方式效率低、主观性强，缺乏跨规划周期的系统对比。本工具将十四五与十五五规划纲要原文按产业标签做语义对齐对比，用高亮标注变化方向，帮助用户快速建立"政策资金流向雷达"。

### 目标用户

- 投资研究员：在政策发布后快速扫描产业倾斜方向，判断资源重新分配趋势
- 个人投资者：理解五年规划的产业变化，辅助建立投资观察池（不做荐股）

---

## 2. 项目目标

让用户在一个页面上完成"选择产业 → 查看原文对比 → 浏览产业标签变化表 → 阅读投资观察要点"的完整流程，建立从政策文本到投资前置研究的认知链路。

---

## 3. 核心功能

### 功能 1：原文对比层

- **描述**：左侧十四五原文 vs 右侧十五五原文，按产业标签对齐展示。AI 标注的变化用颜色高亮：新增/强化（黄色）、弱化（灰色）、工程化（蓝色）。点击高亮文字弹出解释卡片（为什么标这个颜色、对产业链的意义）。
- **优先级**：P0
- **验收标准**：10 个产业标签的原文对比均可正确渲染，高亮颜色正确，点击/Hover 解释卡片正常弹出。

### 功能 2：产业标签对比表

- **描述**：10 行 × 5 列表格，列：产业标签、十四五状态、十五五状态、变化判断、资源倾斜方向。始终展示全表，当前选中行高亮。切换产业标签时表联动更新高亮行。
- **优先级**：P0
- **验收标准**：10 个产业标签全部展示，列数据完整，选中行高亮正确。

### 功能 3：投资观察卡片

- **描述**：选中产业后展示该产业的投资观察要点，字段：政策变化、资源倾斜强度（高/中/低）、落地证据、对应产业链、风险提示。
- **优先级**：P0
- **验收标准**：切换产业后卡片内容联动更新，5 个字段均完整展示。

### 功能 4：产业标签切换

- **描述**：两行网格（每行 5 个），点击标签后页面平滑滚动到原文对比层顶部，三层内容联动更新为所选产业。
- **优先级**：P0
- **验收标准**：点击任意标签后内容正确切换，滚动行为符合预期。

### 功能 5：总体变化摘要

- **描述**：页面顶部显示标题 + 从 10 个产业标签数据中提取的总体变化摘要（2-3 句话概括产业政策重心转移方向）。
- **优先级**：P0
- **验收标准**：摘要内容从 JSON 数据动态生成，非硬编码。

## 4. 不做的事情（边界）

- 不做各省数据（只做全国十四五 vs 十五五）
- 不做 AI API 对接（使用静态 JSON）
- 不做 PDF 上传解析
- 不做用户上传原文
- 不做地区筛选、领域筛选、对比模式切换
- 不做变化排名视图
- 不做多页面路由
- 不做 ETF/股票推荐
- 不做后端服务 / 数据库
- 不做暗色模式切换（仅在系统 prefers-color-scheme 下自动适配）
- 不做多语言

---

## 5. 用户故事

### 故事 1
- **作为** 投资研究员
- **我想要** 在一个页面上看到十四五和十五五的产业政策原文并排对比，变化处高亮标注
- **以便** 快速判断哪些产业获得了更多政策资源倾斜

### 故事 2
- **作为** 个人投资者
- **我想要** 浏览完整的产业标签变化表，并查看每个产业的投资观察要点
- **以便** 将政策变化转化为可跟踪的投资观察方向，而不是盲目追热点

---

## 6. 技术约束

- React + Vite + TypeScript + Tailwind CSS
- 纯静态页面，无后端
- 数据源：`data/industries.json`（静态 JSON）
- 支持桌面端（主要使用场景），同时保证移动端可阅读
- 支持 `prefers-color-scheme` 自动切换浅色/暗色模式
- 不引入 UI 组件库

---

## 7. 风险与假设

| 风险 | 影响 | 应对方式 |
|------|------|---------|
| 静态 JSON 数据被误认为 AI 实时分析结果 | 低 | 页面底部注明数据来源和产出方式 |
| 投资观察内容被误解为投资建议 | 中 | 页面明确标注"仅供研究参考，不构成投资建议" |
| 十五五 PDF 原文获取可能受政策发布节奏影响 | 中 | 已确认国家发改委官网有官方 PDF，版本可控 |
| Tailwind CSS v4 与 Vite 集成可能有配置坑 | 低 | 项目初始化后 Codex 技术复核验证 |
